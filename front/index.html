Источники данных
================

# JIRA

Данный вид источника позволяет создавать метрики на основе данных Attlassian Jira.

## Настройка

Создание и редактирование источника происходит в разделе Data sources / JIRA. Сохранённые источники начинают автоматически индексироваться после создания и становятся доступны для использования в любой метрике.
При создании источника нужно указать логин и пароль, используемый для авторизации в Jira, а также список проектов, которые будут доступны в метриках.
Поле Index period определяет частоту, с которой будет запускаться индексирование источника.

## Индексация

Индексация - это автоматический периодический процесс преобразования сырых данных источника в канонические данные в формате коллекции базы MongoDB, к которым можно обращаться в метрике.
Периодическая индексация начинает происходить автоматически после сохранения источника. Также можно запустить индексацию принудительно, нажав на кнопки Indexify JIRA (индексировать только последние изменения в источнике) или Indexify JIRA [last 3 month] (индексировать все изменения за последние 3 месяца).

Метрики
=======

Метрика - это заданный пользователем алгоритм (правила) обработки индексированных данных источников для дальнейшего отображения на дашбордах. Создание и редактирование метрик происходит в разделе Metrics. Для метрики необходимо указать имя, настроить стадии обработки данных, которых может быть любое количество, но обязательно нужно задать терминальную стадию. Терминальная стадия - это та, результат работы которой выводится на дашборде.
Каждая стадия принимает на вход массив данных из индексированного источника (db) или другой стадии (stage). Откуда стадия получает данные указывается в поле Inputs / Input type. Далее стадия проводит их преобразование и отдаёт на выходе новый массив данных, который переходит в обработку в другой стадии или указывается как терминальный. Каким способом стадия обрабатывает данные указывается в поле Action type. Это поле может принимать следующие значения
  • query
    Стадия применяет ко входящим данным запрос, описываемый синтаксисом MongoDB.

  • rpc
    Стадия выполняет HTTP запрос, передавая в него входящие данные и используя ответ в качестве результата работы стадии.
    Описание протокола работы внешнего метода:

Дашборды
========

Дашборд - это визуально представление результата работы метрик в избранном пользователем виде. Каждый вид представления требует определённого формата результата работы метрики. С ними можно ознакомиться в описании каждого вида.

Виды представлений:
  • table
    Результаты представлены в виде простой таблицы.
    Требования к метрике отсутствуют, любую метрику можно представить в виде таблицы.

  • timeseries
    Представление числовых результатов на временной шкале.
    Требования к метрике: метрика возвращает массив документов с обязательными полями label и value.
    Пример валидного результата работы метрики: 
    [
    {
        "label": "Line 1",
        "value": 10
    },
    {
        "label": "Line 2",
        "value": 1
    }
]

  • gantt
    Результаты представлены в виде диаграммы Гантта.
    Требования к метрике:
    1. Метрика возвращает массив документов с обязательными полями name и data.
    2. Поле name должно содержать текстовое описание задачи.
    3. Поле data должно содержать массив объектов с обязательными полями x и y.
    4. Поле y должно содержать диапазон дат в виде списка вида ["2024-08-19 07:07:26","2024-08-22 07:37:42"].
    5. Поле x должно содержать имя исполнителя.

    Пример валидного результата работы метрики: 
    [
    {
        "name": "TASK-1",
        "data": [
            {
                "y": "Ivan Petrov",
                "x": [
                    "2024-08-26 11:21:02",
                    "2024-08-26 12:19:01"
                ]
            },
            {
                "y": "Lazar Primus",
                "x": [
                    "2024-08-26 13:52:10",
                    "2024-08-26 13:52:21"
                ]
            }
        ]
    }
]
